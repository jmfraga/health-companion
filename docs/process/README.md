# Process documents

Internal artifacts from the build: the founder's thesis, the original
hackathon brief that kicked the project off, the iterative planning
docs, the journals, and the run-through findings. These are not
reference documentation — they are the record of how the product got
to where it is. Kept as an archive so decisions can be re-read with
context intact.

If you're looking for **what the product does today** or **how it is
built**, start at:

- [`../../README.md`](../../README.md) — project overview
- [`../../ROADMAP.md`](../../ROADMAP.md) — product vision and capability threads
- [`../architecture.md`](../architecture.md) — runtime architecture
- [`../agents.md`](../agents.md) — how Opus 4.7 is used
