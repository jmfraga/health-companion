## 3. Recommended hero demo — "Laura's first visit"

### Why this flow
It combines **the three differentiating axes** of the product (longitudinal memory, proactivity, clinical accompaniment) in **3–5 minutes**, with **4 distinct wow moments** and 4 distinct uses of Opus 4.7.

> **Update (April 21)**: in practice the demo was tightened to 2 acts (~45s + ~55s) to match a 3-minute presentation window. The four-act framing below is preserved for historical context. See `bitacora-desarrollo.md` for the acts that ship.

### Demo script (live, in front of the judges)

**Scene 1 · Conversational onboarding (60s)**
- Laura, 44, opens the app. She talks to the app by voice or text: *"Hi, I'm Laura, I'm 44, my mom died of breast cancer at 52."*
- Opus 4.7 extracts clinical entities in real time and builds a structured health profile as Laura speaks. The UI shows the profile **forming live** (age, family history, inferred risks).
- **Wow #1**: the judges see the model *thinking clinically*, not just chatting.

**Scene 2 · Uploading the lab result (60s)**
- Laura drags in a PDF of real (anonymized) lab results. Multimodal.
- Opus 4.7 extracts values, identifies glucose at 118 mg/dL (prediabetes), and cross-references with the profile.
- Response: educates (WHO), contextualizes (diabetic father → no BRCA inheritance, but DM yes), refers (talk to your doctor at your next visit).
- **Wow #2**: multimodal + clinical reasoning + regulatory framing respected.

**Scene 3 · Personalized plan generated by subagents (90s)**
- Laura asks *"help me take care of myself."* The system launches **3 subagents in parallel** (visible on screen):
  - Screening agent: schedules mammography (maternal history), colonoscopy at 45, Pap.
  - Lifestyle agent: nutrition + exercise plan adapted to the profile.
  - Mental health agent: recognizes the emotional load of the maternal history; offers an empathetic check-in.
- Opus 4.7 (orchestrator) synthesizes the three outputs into a coherent, prioritized plan.
- **Wow #3**: real agentic use. Not just an LLM — a multi-agent clinical system.

**Scene 4 · Memory that builds a relationship (60s)**
- Simulate "6 months later." Laura comes back. The app says: *"Hi Laura. Six months ago we talked about your mom. How have you been feeling? I see you finished your mammography — that makes me happy. Your fasting glucose last month was 108, down from 118. Your walking plan is working."*
- The **health timeline** (visual widget) with milestones is shown.
- **Wow #4**: the differentiator that kills ChatGPT Health — *a real longitudinal relationship.*

### Why the judges will remember it
- **Human**: Laura, not "the user." The story hooks.
- **Medically real**: the reasoning maternal history → early screening is clinically valid (Dr. Fraga can defend it under any question).
- **Technically surprising**: multimodal + subagents + persistent memory in a single demo.
- **Grounds the concept**: a skeptical judge stops seeing "just another health app" and sees a product.

## 4. Exact MVP scope (scope freeze)

### Build
| Component | Detail | Priority |
|-----------|--------|----------|
| Conversational chat UI | Text (voice if time allows). Must *feel* like a doctor, not like ChatGPT. | P0 |
| Live structured-profile extractor | Side panel that fills in as the user speaks. | P0 |
| Multimodal lab PDF ingestion | One well-tested sample PDF. Not ten. | P0 |
| Orchestrator with 3 subagents | Screening, Lifestyle, Mental health. | P0 |
| Persistent memory (demo-simulated) | Store profile + history in JSON / SQLite. "6 months later" demo uses a pre-loaded fixture. | P0 |
| Visual health timeline | Simple React component with milestones. | P1 |
| Voice (input / output) | Only if there is time. Raises the "cool to watch" quotient a lot. | P2 |

### Do not build
- Login, auth, multi-user → use a single seed profile.
- Real push notifications → show a mock of the message.
- Multi-language onboarding → only Spanish for the demo (or English; pick one).
- Integration with Apple Health / wearables → outside the MVP.
- Native apps → PWA / web.
- Payments / freemium → outside the MVP.
- The full five pillars → only those needed for the hero flow.

## 5. Use of Opus 4.7 (20% of the score — key differentiator)

The judges come from Anthropic. A chat wrapper disappoints them. We must show **Opus-4.7-specific capabilities** that don't come out of a basic prompt.

### Capabilities to demonstrate explicitly

1. **Extended thinking visible in a clinical UI**
   - Show (partially) the model's reasoning when evaluating a lab result. Example: a "See reasoning" disclosure that reveals the educational differential. Clinical transparency is unique.

2. **Multimodal over a real lab PDF**
   - Do not parse the PDF with libraries: let Opus 4.7 read the PDF directly and extract the values in structured JSON with confidence.

3. **Multi-agent orchestration with specialist subagents**
   - Use the SDK to launch 3 child agents in parallel, each with a specialist medical system prompt. The orchestrator consolidates. **This is what surprises.**

4. **Real tool use (not decorative)**
   - Tools: `save_profile_field`, `schedule_screening`, `fetch_recommendations_for_age_sex`, `log_biomarker`. The model decides when to call them during the conversation.

5. **Long-term memory managed by the model**
   - Do not save "the whole history" — use Opus to *decide what is worth remembering* and write to a structured memory table (episodic vs. semantic). Show the memory grow.

6. **Reinforced medical guardrail as a system prompt**
   - Never diagnoses, never prescribes, always refers. This is a real regulatory requirement (FDA wellness / COFEPRIS compliance). Showing the model respect it even when the user insists → **impresses on craft**.

### What does NOT impress
- *"We put a prompt on Claude and asked it about symptoms"* → 0 creative points.
- *"Fine-tuning"* → no time and not needed (the author's 480+ cases already prove the base model is enough).

## 6. Minimum recommended architecture

```
┌─────────────────────────────────────────────────────────┐
│  Frontend (Next.js + React + Tailwind + shadcn)         │
│  - Chat UI                                               │
│  - Live profile panel (populated via tool-use events)   │
│  - Health timeline (React component)                     │
│  - PDF upload drop-zone                                  │
│  - Voice input (Web Speech API optional)                 │
└─────────────────────────────────────────────────────────┘
                         │  SSE / WebSocket
┌─────────────────────────────────────────────────────────┐
│  Backend (Python FastAPI)                                │
│  - /api/chat        → orchestrator with Opus 4.7 + tools │
│  - /api/ingest-pdf  → multimodal Opus 4.7                │
│  - /api/plan        → fan-out to 3 subagents             │
│  - /api/memory      → profile read / write               │
└─────────────────────────────────────────────────────────┘
                         │
┌─────────────────────────────────────────────────────────┐
│  Storage                                                 │
│  - SQLite (enough for the demo; no Postgres)            │
│  - Tables: profile, episodic_memory, biomarkers, plan    │
└─────────────────────────────────────────────────────────┘
                         │
┌─────────────────────────────────────────────────────────┐
│  Claude Agent SDK + Opus 4.7                             │
│  - Main agent (doctor-companion)                         │
│  - Subagents: screening, lifestyle, mental-health        │
│  - Tools: save_profile_field, log_biomarker,             │
│           schedule_screening, fetch_guidelines           │
└─────────────────────────────────────────────────────────┘
```

### Defensible technical decisions
- **Next.js + FastAPI**: fast to prototype and easy to demo.
- **SQLite**: zero infra overhead on demo day.
- **Agent SDK for subagents**: shows the team knows how to use the official Anthropic tools.
- **SSE streaming**: the judges see the model work live, not a spinner.

## 8. Risks and mitigations

| Risk | Probability | Mitigation |
|------|-------------|------------|
| The demo breaks live (network, API) | High | Backup video + local fixtures + SQLite + ability to run a local fallback model (the author already has the infra). |
| The judges ask "what about regulation?" | High | 20s slide / speech: *"Categorized as wellness, not a medical device. Endorsed by a physician. Meets FDA / COFEPRIS / MDR guidelines."* See `concept-v1.md` §Regulatory framework. |
| *"Why not just ChatGPT Health?"* | Very high | Rehearsed answer: *"ChatGPT is reactive — you ask. We reach out. And we have longitudinal clinical memory."* See `competitive-analysis-v1.md`. |
| The model hallucinates lab values | Medium | Force structured output with schema + validation. Show confidence per value. Re-ask on ambiguity. |
| The subagents take too long | Medium | Aggressive timeout + skeleton loader + real parallel fan-out (not sequential). |
| Lack of visual polish | Medium-high | Use shadcn + lucide-react. Do not design from scratch. Two hours on typography + spacing are worth more than one extra feature. |
